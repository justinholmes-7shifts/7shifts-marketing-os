# Campaigns

One file per campaign. This is the running log: the brief, the working notes, the assets, and the outcome. Think of each file as a session summary for that campaign.

**Naming:** `YYYY-QN-campaign-name.md` (e.g. `2026-q3-summer-labor-cost.md`)

**When a campaign wraps:** pull the durable lessons up into `../learnings.md` so the next person inherits them. The campaign file is the firehose; `learnings.md` is the curated takeaway.

**Suggested structure for a campaign file:**

```
# [Campaign name]

## Goal
What we're trying to move, and the number.

## Audience
Which ICP segment (link to ../context/icp.md).

## Message
The core angle (pull from ../context/positioning.md and ../context/voice.md).

## Assets
Links / drafts.

## Results
What happened. Real numbers.

## Learnings
What we'd repeat, what we'd change. Promote the durable ones to ../learnings.md.
```
